CLAUDE CODE 入門

これだけ覚えれば、
Claude Codeは怖くない。

Claude Codeを使いこなすために「必ず覚えたい言葉」と「動く仕組み」を、図解とたとえ話で1ページにまとめました。上から順に読めば、専門知識ゼロでも全体像がつかめます。

このページの使い方
本文中の点線つきの言葉にカーソルを乗せる（スマホはタップ）と、その場で意味が表示されます。すべての用語は巻末の用語集にもまとまっています。各章末の「ここだけは覚える」だけ拾い読みしてもOKです。

Claude Codeとは何か

まずは「チャットAIと何が違うのか」から。ここが一番大事な出発点です。

Claude Code（クロード コード）は、Anthropic（アンソロピック）社のAI「Claude」が、あなたのパソコンの中で実際に手を動かして作業してくれるツールです。

ChatGPTのような「チャットAI」は、質問に言葉で答えることしかできません。一方Claude Codeは、ファイルを読み書きし、コマンドを実行し、調べものをし、結果を確認しながら作業そのものを完了まで進めます。このように自分で考えて行動するAIをエージェントと呼びます。

たとえるなら

チャットAIは「電話の向こうのアドバイザー」。やり方は教えてくれますが、作業するのはあなたです。Claude Codeは「隣の席に座って実際にPCを操作してくれるアシスタント」。指示を出せば、手を動かすところまでやってくれます。

どこで使えるのか

基本の形

ターミナル（CLI）

黒い画面に文字で指示を打ち込む形。ターミナルで claude と入力すると起動します。一番シンプルで、すべての機能が使えます。

おすすめ

エディタ拡張（VS Codeなど）

コードやファイルを見ながら横のパネルで会話できる形。ファイルの変更内容が色つきで見えるので、初心者はここから始めるのがおすすめです。

その他

デスクトップアプリ

Mac / Windows向けの専用アプリ。ターミナルを開かなくても複数の作業を並行して進められます。

その他

Web版（claude.ai/code）

ブラウザから使う形。自分のPCではなくクラウド上の環境で作業させることができます。

「コードを書く人専用」ではない

名前に「Code」とありますが、実際にできることはプログラミングに限りません。ファイル整理、文書作成、データ集計、リサーチ、定型業務の自動化など、「パソコンの上で行う作業」全般を任せられます。経営者やビジネスパーソンにとっては「業務を自動化してくれる万能アシスタント」と捉えるのが正確です。

ここだけは覚える
  • Claude Code ＝ 実際に手を動かすAIアシスタント（答えるだけのチャットAIとは別物）
  • 自分で考えて行動するAIを「エージェント」と呼ぶ
  • プログラミング専用ではなく、PC上の作業全般を任せられる

動く仕組み — エージェントループ

指示を出したあと、Claude Codeの中で何が起きているのか。この流れが分かると挙動が読めるようになります。

Claude Codeに指示を出すと、内部では次のサイクルが回ります。これをエージェントループと呼びます。

STEP 1指示を受けるあなたが日本語で依頼（＝プロンプト）
→
STEP 2考える何をすべきか計画を立てる
→
STEP 3ツールを使うファイルを読む・書く・コマンド実行・検索
→
STEP 4結果を確認うまくいったか自分で検証
→
STEP 5完了を報告結果をあなたに説明
完了するまで STEP 2〜4 を何十回でも繰り返す
↑ 完了するまで STEP 2〜4 を繰り返す ↑

「ツール」＝ Claudeの手足

AIの本体（LLM）は、本来「文章を作る」ことしかできません。Claude Codeはそこにツールという「手足」を与えることで、実際の作業を可能にしています。主なツールは次の通りです。

ツール名できること身近に言うと
Readファイルの中身を読む書類を開いて読む
Write / Editファイルを新規作成・修正する書類を書く・直す
Bashコマンドを実行するパソコンに直接指示を出す
Glob / Grepファイルを探す・中身を検索する書庫から目当ての書類を探す
WebSearch / WebFetchWeb検索・ページの読み込みネットで調べものをする
Task（サブエージェント）別のClaudeに作業を任せる部下に仕事を振る（第10章）
たとえるなら

優秀な新人に仕事を頼んだ場面と同じです。「資料まとめておいて」と言えば、新人は資料を探し、読み、下書きし、見直して、完成したら報告に来ます。途中の一つひとつの動作（ツール実行）をあなたが指示する必要はありません。ただし大事な判断（危ない操作）のときだけ「これやっていいですか？」と確認に来ます。それがパーミッションの仕組みです。

ここだけは覚える
  • 1回の指示で、Claudeは何十回もツールを実行しながら自走する
  • ツール ＝ AIの手足（読む・書く・実行する・探す）
  • 危ない操作の前だけ許可を求めてくる（勝手に何でもするわけではない）

トークンとコンテキスト — AIの「記憶」の仕組み

「なぜ長い会話は苦手なの？」「なぜ毎回CLAUDE.mdを読むの？」の答えがここにあります。

トークン — AIが文章を数える単位

トークンとは、AIが文章を処理するときの最小単位です。おおまかに日本語1文字 ≒ 1〜2トークン、英単語1つ ≒ 1トークン強。AIの利用料金も「何トークン処理したか」で決まります。

コンテキストウィンドウ — 作業机の広さ

コンテキストウィンドウとは、Claudeが一度に「見えている」情報の上限です。モデルにより約20万〜100万トークン（日本語の書籍にして数冊〜十数冊分）。会話の履歴・読んだファイル・ツールの実行結果は、すべてこの中に載っています。

たとえるなら

コンテキストウィンドウは「作業机の広さ」です。机に載っている書類（＝今の会話・読んだファイル）しか見えません。作業が長引くと机が書類でいっぱいになり、能率が落ちます。そして翌朝出社すると机は完全に空（＝新しいセッションでは前の会話を覚えていない）。だから「毎朝必ず机に置いておく申し送りノート」が必要になります。それがCLAUDE.mdです。

机を片付ける2つのコマンド

要約して続ける

/compact

それまでの会話を要約して圧縮し、要点だけ机に残して作業を続けます。長い作業の途中で机が狭くなってきたときに使います。放っておいても、上限が近づくと自動で要約されます（auto-compact）。

全部片付ける

/clear

会話履歴を完全にリセットして、まっさらな机で始めます。別の仕事に切り替えるときはこちら。前の作業内容は引き継がれません。

実務のコツ：1つの仕事が終わったら /clear する習慣をつけると、Claudeの精度が安定します。関係ない古い話題が机に残っていると、判断がぶれる原因になります。
ここだけは覚える
  • トークン ＝ AIが文章を数える単位（料金と記憶量の単位）
  • コンテキストウィンドウ ＝ 作業机の広さ。机の上の物しか見えない
  • セッションが変わると記憶はゼロ。だからCLAUDE.mdに書き残す
  • /compact ＝ 要約して続行、/clear ＝ 完全リセット

モデル — Claudeの「頭脳」を選ぶ

同じClaudeでも、中身の頭脳にはグレードがあります。使い分けを覚えると、速さと質とコストのバランスが取れます。

モデルとは、AIの頭脳そのもののこと。Claudeには性格の異なる複数のモデルがあり、Claude Codeの中では /model というスラッシュコマンドでいつでも切り替えられます。

最高性能

Opus（オーパス）

いちばん賢いモデル。複雑な設計、難しい調査、重要な意思決定の壁打ちなど「じっくり考えてほしい仕事」向け。

遅め・高コスト。ここぞの場面で。

バランス型

Sonnet（ソネット）

賢さと速さのバランスが良い主力モデル。日常の作業の大半はSonnetで十分こなせます。迷ったらこれ。

日々の主力。コスパ良好。

高速・軽量

Haiku（ハイク）

軽くて速いモデル。単純な整形、簡単な質問、大量の繰り返し処理など「量をこなす仕事」向け。

最速・最安。単純作業に。

各モデルには「Opus 4.8」「Haiku 4.5」のように世代番号がつきます。数字が大きいほど新しい世代です。2026年には最上位の新ティアとしてClaude 5ファミリー（Fable 5 / Mythos 5）も登場しました。名前の格付け（賢い順）は「Fable ＞ Opus ＞ Sonnet ＞ Haiku」と覚えておけば十分です。

たとえるなら

社内の人材配置と同じです。エース（Opus）に全部の仕事を振ると高くつくし待ち時間も長い。日常業務は頼れる主力（Sonnet）、雑務はフットワークの軽い若手（Haiku）。仕事の重さで人を選ぶように、モデルを選びます。

ここだけは覚える
  • 賢い順に Opus ＞ Sonnet ＞ Haiku（最上位に Claude 5 / Fable）
  • 賢いほど遅くて高い。仕事の重さでモデルを選ぶ
  • 切り替えは /model

フォルダ構造 — 設定はどこに置かれているか

Claude Codeの設定・記憶・拡張機能は、決まった場所のファイルで管理されます。場所は2つだけ覚えれば大丈夫です。

覚える場所は2つ。ホームの ~/.claude/（あなた個人の設定・全プロジェクト共通）と、各プロジェクト内の .claude/（そのプロジェクト専用・チームで共有できる）です。

先に用語を2つ：ホームディレクトリの「~（チルダ）」は自分のホームフォルダの略記号。「.claude」のように名前がドットで始まるファイル・フォルダは隠しファイルといい、普段は画面に表示されない設定用の置き場所です。
~/.claude/ ← あなた個人の設定（全プロジェクト共通） ├── CLAUDE.md ← 全プロジェクトで毎回読まれる指示書 ├── settings.json ← 個人設定（許可ルール・モデルなど） ├── commands/ ← 自作スラッシュコマンド置き場 ├── agents/ ← サブエージェント定義置き場 └── skills/ ← スキル置き場 あなたのプロジェクト/ ← 仕事ごとのフォルダ ├── CLAUDE.md ← このプロジェクト専用の指示書（チーム共有） ├── .claude/ │ ├── settings.json ← プロジェクト設定（チーム共有・git管理） │ ├── settings.local.json ← 自分だけの設定（共有されない） │ ├── commands/ agents/ skills/ ← プロジェクト専用の拡張 └── .mcp.json ← 外部サービス接続の設定（第11章）
たとえるなら

~/.claude/ は「自分の手帳」。どの現場に行っても持ち歩く自分ルール。プロジェクトの .claude/ は「現場の掲示板」。その現場に入る人全員（チームメンバー全員のClaude）が同じルールを見ます。

同じ設定がぶつかったら？ — 優先順位

複数の場所に同じ設定があるときは、次の順で強い方が勝ちます。

強い（優先される）弱い
1
企業の管理ポリシー
会社のIT部門が配布する設定（個人では変更不可）
2
起動時の指定（コマンドライン引数）
その場かぎりの一時指定
3
.claude/settings.local.json
プロジェクト内・自分だけの設定
4
.claude/settings.json
プロジェクト共有の設定
5
~/.claude/settings.json
個人のグローバル設定

ざっくり「より狭い範囲・より具体的な指定ほど強い」と覚えればOKです。

ここだけは覚える
  • 設定の場所は2つ：~/.claude/（個人・全体共通）と プロジェクトの .claude/（現場専用）
  • 手帳（個人）と現場の掲示板（プロジェクト）の関係
  • ぶつかったら「狭い範囲の設定が勝つ」

CLAUDE.md — 毎回読まれる「申し送りノート」

Claude Codeを育てる一番の急所。ここを整えるかどうかで成果が大きく変わります。

第3章の通り、Claudeはセッションが変わると何も覚えていません。そこで、起動のたびに必ず読み込まれるMarkdownファイルが用意されています。それが CLAUDE.md（クロード エムディー）です。ここに書いたことは「毎回の会話の前提知識」になります。

3つの階層で読み込まれる

場所効く範囲書く内容の例
~/.claude/CLAUDE.mdすべてのプロジェクト自分の好み・共通ルール（「回答は日本語で」「ファイル削除は必ず確認」）
プロジェクト/CLAUDE.mdそのプロジェクト全体事業の説明・フォルダの意味・よく使うコマンド・命名ルール
サブフォルダ/CLAUDE.mdそのフォルダで作業するときその部門・案件だけの細かいルール

上の階層から順にすべて読み込まれ、重ね合わせて適用されます。「会社全体の就業規則 → 部門のルール → 案件ごとのメモ」という重なり方です。

書き方のコツ

  • 短く・具体的に・箇条書きで。長文の物語より「・請求書はoutput/に保存」のような1行ルールが効きます
  • 「するな」より「こうして」。禁止だけでなく正しいやり方をセットで書く
  • 会話の途中で「これ覚えておいて」と頼むか、メッセージの先頭に # を付けて送ると、その内容をCLAUDE.mdに追記してくれます
  • /init と打つと、プロジェクトを分析してCLAUDE.mdの初版を自動で作ってくれます
  • @ファイルパス と書くと、別ファイルの内容を取り込めます（ルールを別ファイルに分けて整理できる）
プロジェクト/CLAUDE.md の例
# このプロジェクトについて
- 事業内容: キックボクシングジムの運営
- 顧客向け文書は英語、社内メモは日本語で書く

# ルール
- 生成したファイルは output/ フォルダに保存する
- 表計算が必要なときはGoogle Sheetsを使う（Excel禁止）
- 削除の前には必ず確認を取る

# 詳細ルールの読み込み
@rules/writing-style.md
たとえるなら

CLAUDE.mdは「新しく来たアシスタントに毎朝渡す申し送りノート」。Claudeは毎回“初出勤”なので、ノートが充実しているほど初日から即戦力になります。逆にノートが空だと、毎回ゼロから説明することになります。

ここだけは覚える
  • CLAUDE.md ＝ 起動のたびに必ず読まれる申し送りノート
  • 個人（全体）→ プロジェクト → サブフォルダ の3階層で重なる
  • 短い箇条書きが最強。/init で初版を自動生成できる

設定とパーミッション — 安全装置の仕組み

「AIが勝手に何かしたら怖い」への答え。Claude Codeには多層の安全装置が組み込まれています。

Claude Codeは、ファイルの変更やコマンド実行のような「影響のある操作」の前にあなたの許可を求めます。この許可の仕組みをパーミッションと呼びます。毎回聞かれるのは面倒なので、settings.jsonにルールを書いて「これは自動でOK」「これは毎回確認」「これは絶対禁止」を事前に決めておけます。

3段階のルール

段階意味例
allow（許可）確認なしで自動実行してよいテストの実行、ファイルの読み取り
ask（確認）実行前に毎回あなたに確認する外部への送信、機密ファイルの読み取り
deny（禁止）何があっても実行させないメール送信、強制削除、パスワードファイルの読み取り
.claude/settings.json の例
{
  "permissions": {
    "allow": ["Bash(npm run test:*)", "Read(./docs/**)"],
    "ask":   ["Bash(git push:*)"],
    "deny":  ["Read(./.env)", "Bash(rm -rf:*)"]
  }
}

ルールは「ツール名（対象のパターン）」という形で書きます。上の例なら「npm run test で始まるコマンドは自動OK」「git push は毎回確認」「.envファイルは読むこと自体を禁止」という意味です。同じ操作に複数のルールが当てはまる場合は、deny ＞ ask ＞ allow の順（厳しい方が勝つ）で判定されます。

パーミッションモード — セッション全体の構え

標準

default

初期状態。安全な読み取りは自動、変更系は初回に確認。迷ったらこれ。

編集は自動

acceptEdits

ファイル編集の確認を省略して自動承認。作業のテンポが上がる分、変更をあとでまとめて見る前提で。

計画だけ

plan（プランモード）

読み取り専用のモード。ファイルを一切変更せず、調査と計画立案だけさせる。大きな作業の前に「まず計画を見せて」ができる。Shift+Tabで切替。

上級者向け

bypassPermissions

すべての確認をスキップ。隔離された安全な環境以外では使わないこと。

新しいバージョンには、AIが操作の中身を見て安全なものだけ自動承認する「auto」モードなども追加されています。まずは上の4つ（特に default と plan）を覚えれば十分です。

たとえるなら

新しいアシスタントへの決裁権限の設計と同じです。「コピー取りは勝手にやっていい（allow）」「取引先へのメールは上司確認（ask）」「契約書への押印は絶対に自分でやらない（deny）」。権限設計を最初に決めておけば、あとは安心して任せられます。

ここだけは覚える
  • パーミッション ＝ Claudeの決裁権限ルール（allow / ask / deny の3段階）
  • ルールは settings.json に書いて事前に決められる
  • プランモード（読み取り専用）を使えば「まず計画だけ」が安全にできる

Hooks — 決まったタイミングで必ず動く自動処理

「AIにお願いする」のではなく「仕組みで必ず実行させる」ための機能です。

Hooks（フック）とは、Claude Codeの動作の節目（イベント）に引っかけて、自分で決めた処理を自動実行させる仕組みです。CLAUDE.mdに「毎回◯◯して」と書いてもAIが忘れる可能性はゼロではありません。Hooksはプログラムとして必ず実行されるので、忘れることがありません。

たとえるなら

CLAUDE.mdのルールが「口頭の指示」だとすると、Hooksは「ドアに付けたセンサー」。ドアが開けば（＝イベントが起きれば）必ずチャイムが鳴ります。人の記憶に頼らない、確実な自動化です。

主なイベント（発火するタイミング）

イベント名いつ動くか使い道の例
PreToolUseツール実行の直前危険なコマンドを検査してブロックする
PostToolUseツール実行の直後ファイル編集のたびに自動で整形ツールをかける
UserPromptSubmitあなたが指示を送信した時毎回の指示に定型の注意書きを差し込む
StopClaudeが応答を終える時作業完了を通知音やチャットで知らせる
SubagentStopサブエージェントの作業完了時部下エージェントの結果を記録する
PreCompact会話の要約（compact）の直前要約前に大事なメモを退避する
SessionStart / SessionEndセッションの開始時 / 終了時開始時に最新情報を読み込ませる
NotificationClaudeが通知を出す時（許可待ちなど）スマホやSlackに転送する

このほかにも、ツールが失敗した時（PostToolUseFailure）、許可を求められた時（PermissionRequest）、監視中のファイルが変わった時（FileChanged）など、多くのイベントが用意されています。まずは上の表の代表的なものを覚えれば十分です。

設定のイメージ

settings.json に書くHooksの例（編集のたびに自動整形）
{
  "hooks": {
    "PostToolUse": [{
      "matcher": "Edit|Write",          ← どのツールの時に動くか
      "hooks": [{ "type": "command",
                   "command": "npx prettier --write ..." }]
    }]
  }
}
実例：「削除コマンド rm を検知したら、復元できるゴミ箱行きコマンドに自動で置き換える」という安全フックを PreToolUse に仕込む、といった使い方ができます。AIの善意に頼らず、仕組みで事故を防げるのがHooksの価値です。
ここだけは覚える
  • Hooks ＝ 節目のイベントに引っかけて必ず実行される自動処理
  • CLAUDE.md（お願い）と違い、忘れることがない
  • 代表イベント：PreToolUse（直前・ブロック可）/ PostToolUse（直後）/ Stop（応答完了時）

スラッシュコマンド — よく使う操作の短縮ダイヤル

「/」から始まる命令を覚えると、Claude Codeの操作が一気に速くなります。自作もできます。

スラッシュコマンドは、チャット欄に / から始まる短い命令を打つ機能です。よく使う組み込みコマンドは次の通り。まずは太字の6つを覚えれば十分です。

コマンド何をするか
/help使えるコマンドの一覧を表示する
/initプロジェクトを分析してCLAUDE.mdを自動生成する
/clear会話を完全リセットする（新しい机で開始）
/compact会話を要約して圧縮する（机を片付けて続行）
/modelモデル（Opus / Sonnet / Haiku など）を切り替える
/permissions許可ルール（allow / ask / deny）を確認・変更する
/memoryCLAUDE.md（記憶ファイル）を開いて編集する
/config設定画面を開く（テーマ・通知など）
/mcpMCP接続（第11章）の状態を確認・管理する
/agentsサブエージェント（第10章）を確認・作成する
/hooksHooks（第8章）を対話形式で設定する
/contextコンテキスト（作業机）の使用状況を確認する
/cost今のセッションで使ったトークン量・料金を確認する
/resume過去のセッション（会話）を選んで再開する
/rewind会話やファイルの状態を少し前に巻き戻す
/statusログイン状態・モデル・接続などの現状を確認する

自作コマンド — 定型業務を1語に圧縮する

.claude/commands/ フォルダにMarkdownファイルを置くだけで、自分専用のコマンドが作れます。ファイル名がそのままコマンド名になります。

.claude/commands/weekly-report.md → 「/weekly-report」で呼び出せる
今週の作業ログを集計して、週報を作成してください。

手順:
1. output/logs/ の今週分のファイルを読む
2. 実施した作業を3〜5項目に要約する
3. output/weekly/ に「YYYY-MM-DD_週報.md」として保存する

補足の指示があれば考慮する: $ARGUMENTS

$ARGUMENTS と書いた場所には、コマンドの後ろに添えた言葉（例：/weekly-report 先方向けに丁寧に）が差し込まれます。

補足：新しいバージョンでは、自作コマンドは次章の「スキル」と同じ仕組みに統合されつつあります（.claude/skills/ 形式が推奨）。どちらの置き方でも「/名前」で呼び出せる点は同じです。
たとえるなら

スラッシュコマンドは「電話の短縮ダイヤル」。毎回長い依頼文を打つ代わりに、よくある依頼を1語に登録しておくイメージです。

ここだけは覚える
  • まず覚える6つ：/help /init /clear /compact /model /permissions
  • 自作コマンドは .claude/commands/名前.md を置くだけ
  • 定型業務は自作コマンドに圧縮するのが上達の近道

スキルとサブエージェント — Claudeに「専門性」を持たせる

1人のClaudeに全部やらせるのではなく、マニュアルを整備し、専門の部下を持たせる段階です。

スキル — 仕事のマニュアル一式

スキルは、特定の仕事のやり方をまとめたマニュアルのパッケージです。.claude/skills/スキル名/SKILL.md という形で、手順書に加えて参考資料や実行用スクリプトも同じフォルダに同梱できます。Claudeは会話の内容から「今この仕事だからこのマニュアルが要る」と判断し、必要なときだけ自動で読み込みます。

.claude/skills/invoice/SKILL.md の例
---
name: invoice
description: 請求書を作成するときの手順。金額・宛先の確認ルールを含む。
---

# 請求書作成の手順
1. templates/invoice-template.md の書式を使う
2. 宛先と金額は必ず口頭確認してから確定する
3. 完成したら output/invoices/ に保存する

サブエージェント — 専門の部下を持つ

サブエージェントは、メインのClaudeが呼び出せる別のClaude（部下）です。.claude/agents/名前.md に「役割・使えるツール・使うモデル」を定義しておくと、メインのClaudeが適材適所で仕事を振ります。

.claude/agents/reviewer.md の例
---
name: reviewer
description: 文書や成果物の品質チェック担当。誤字・数字の整合・言い過ぎを検査する。
tools: Read, Grep, Glob      ← 読み取り専用の部下にできる
---

あなたは品質チェックの専門家です。
渡された成果物を批判的に検査し、問題点を重大度順に報告してください。

サブエージェントの2大メリット

  • 机（コンテキスト）を汚さない — 部下は自分専用の作業机で働くため、大量の調べものをしてもメインの机は要点の報告だけで済む
  • 並列で働ける — 複数の部下に同時に別の仕事を振れるので、大きな作業が速く終わる
たとえるなら

スキルは「棚に整備された業務マニュアル」（必要な時に取り出して読む）。サブエージェントは「専門部署への外注」（調査部・校閲部に仕事ごと渡して、報告だけ受け取る）。コマンド（第9章）が「短縮ダイヤル」なのに対し、こちらは組織づくりに近い機能です。

ここだけは覚える
  • スキル ＝ 自動で読まれる業務マニュアル（SKILL.md ＋ 付属資料）
  • サブエージェント ＝ 専用の机を持つ部下Claude（agents/名前.md）
  • 部下に任せるとメインの机（コンテキスト）が汚れない・並列で速い

MCP — 外部サービスとつながる共通規格

GmailもSlackもデータベースも。Claudeの手が「PCの外」に届くようになる仕組みです。

MCP（Model Context Protocol／モデル・コンテキスト・プロトコル）は、AIと外部サービスをつなぐための世界共通規格です。Anthropicが公開し、いまや業界標準になっています。MCPでつなぐと、ClaudeはGmailの検索、Slackへの投稿、データベースの照会、カレンダー登録などを「ツール」として直接実行できるようになります。

Claude CodeあなたのAIアシスタント
↔
MCPサーバーサービスごとの変換アダプタ
↔
GmailSlackGoogle Drive NotionデータベースGitHub社内システム
たとえるなら

MCPは「USB-Cのような共通コネクタ」です。昔は機器ごとに専用ケーブルが必要でしたが、規格が統一されて何でも同じ差し込み口でつながるようになりました。MCPサーバーは各サービス用の「変換アダプタ」にあたります。

使い方の基本

  • 追加はターミナルで claude mcp add、確認・管理は会話中に /mcp
  • 接続情報はプロジェクトの .mcp.json に保存すればチームで共有できる
  • 接続には「自分のPC内でアダプタを動かす方式（ローカル）」と「サービス側が用意したアダプタにネット経由でつなぐ方式（リモート）」がある
注意：MCPサーバーはClaudeに強い権限を与えます。提供元が信頼できるものだけ接続してください。また、初回接続時は認証（そのサービスへのログイン許可）が必要です。
ここだけは覚える
  • MCP ＝ AIと外部サービスをつなぐ共通規格（USB-Cのようなもの）
  • MCPサーバー ＝ サービスごとの変換アダプタ
  • つなげば Gmail・Slack・DB などがClaudeの「ツール」になる
